## What is DevOps?

DevOps is a combination of two words:Development (Dev) + Operations (Ops) = DevOps!
It is a procedure within which software developers and IT operations.
teams work TOGETHER not WITHOUT one another.
A primary goal of DevOps is to get software to market:
With improved quality and fewer defects.



## How DevOps Improves Software Delivery

### 1. Faster Delivery
Traditionally the process of releasing software would take weeks or months.DevOps focuses on Continuous Integration (CI) and Continuous Delivery (CD) pipelines, which enable teams to adopt and leverage DevOps best practices.and Continuous Delivery to automatically build, test and release software. This implies new features and updates "Such as Google, Facebook and Twitter"
connect directly with the target customers in a faster manner.

### 2. It is possible to get fewer errors with higher-quality work.
With DevOps, bugs can be discovered much later and it is expensive to correct.In DevOps automated testing is practiced throughout the development cycle.This will allow bugs to be caught early and addressed promptly,resulting in better quality software.

### 3. Better Team Collaboration
Traditionally, devs and ops ran in silos.They blamed all the problems on the other one when it were bad.DevOps It brings both teams to one place with common Responsibility.Communications and team work are enhanced and geared towards a common purpose.

### 4. Continuous Monitoring
Devops teams continuously monitor their applications in real time, 24/7.Alerts are triggered immediately in event of any problem.This way, issues are addressed before they are noticed by the user.This ensures greater reliability and satisfaction of the users.

### 5. Infrastructure is Code (IaC).
Prior to now, the servers were even installed manually, leading to inaccuracies.DevOps' environments and servers are deployed through code.This makes everything consistent, repeatable and error free.Also, it ensures that the setup is version controlled, like software.
